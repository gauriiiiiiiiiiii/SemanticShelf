SemanticShelf 

SemanticShelf is a semantic book recommendation application. It takes a natural-language query (topic, theme, genre, idea) and returns a ranked list of books whose descriptions and metadata are semantically similar to the query. The system uses sentence-level embeddings, stores them in a local persistent vector database, and serves a web UI with Streamlit.

Key Goals
- Provide meaning-based search (semantic retrieval), not just keyword matching.
- Keep indexing deterministic and persistent across restarts.
- Use environment-driven configuration with validation.
- Offer a professional, responsive, full-viewport UI.

Dataset Details (data/books.csv)
- Format: CSV.
- Required columns:
  - title
  - authors
  - categories
  - description
  - isbn13
- Optional column:
  - subtitle
- Additional columns may exist (isbn10, thumbnail, published_year, ratings, etc.), but only the required/optional columns are used in indexing.
- Preprocessing:
  - Missing columns are created (optional fields become empty strings).
  - Nulls are replaced with empty strings.
  - Whitespace is stripped.
  - A combined text field is built as:
    title + authors + categories + description
  - Rows with empty title are removed.

Core Components and Responsibilities

1) Configuration (src/config.py)
- Loads environment variables once at startup.
- Converts boolean and integer values safely.
- Validates critical constraints:
  - batch size > 0
  - default results > 0
  - max results > 0
  - default results <= max results
  - collection name not empty
  - embedding model id not empty

2) Data Loading (src/data_loader.py)
- Ensures the CSV exists.
- Checks for required columns.
- Normalizes columns to strings and trims whitespace.
- Builds the combined semantic text field.
- Filters out invalid rows.

3) Embedding Model (src/embeddings.py)
- Loads SentenceTransformer once using LRU cache.
- Optional cache folder is supported.
- Sets TOKENIZERS_PARALLELISM=false to reduce noisy logging or oversubscription.

4) Vector Store (src/vector_store.py)
- Creates a persistent ChromaDB client.
- Disables anonymized telemetry.
- Gets or creates the configured collection.
- Provides a reset function to rebuild the collection.

5) Recommender Service (src/recommender.py)
- BookRecommender is the main service.
- Thread-safe lazy initialization with a lock.
- Builds deterministic IDs based on isbn13, handling duplicates.
- Rebuilds the index if the collection count mismatches dataset size.
- Encodes books in batches for efficient indexing.
- Converts distances to scores (bounded relevance score).
- Exposes:
  - initialize(force_reindex=False)
  - recommend_books(query, n_results)

6) UI and App Entry (app.py)
- Defines the Streamlit UI layout and styling.
- Provides a hero section, search input, and results grid.
- Uses safe HTML escaping to render data.
- Adds loading, empty, and error states.
- Shows categories and relevance score on each result card.
- Initializes the recommender at startup.

Functional Flow (Step-by-Step)

Startup Flow
1. Application starts by running `streamlit run app.py`.
2. Logging is configured (src/logging_utils.py).
3. recommender_service.initialize() is called.
4. load_books() loads and preprocesses the CSV dataset.
5. load_embedding_model() loads the SentenceTransformer.
6. get_collection() opens the ChromaDB collection.
7. The collection count is compared to dataset row count.
8. If count mismatches or force_reindex is true:
   - The collection is reset.
   - Books are embedded in batches.
   - Embeddings and metadata are added to Chroma.
9. The app UI is built and the server starts.

Query Flow
1. User types a query and clicks Search (or presses Enter).
2. UI shows a loading state.
3. recommender_service.recommend_books() runs:
   - Validates the query.
   - Clamps result count to available records and config bounds.
   - Embeds the query.
   - Runs Chroma similarity search.
   - Converts distances to relevance scores.
   - Returns Recommendation objects.
4. UI renders result cards with title, author, categories, score, and description.
5. Status message updates to success or error.

Error Handling
- Missing dataset file raises FileNotFoundError at load time.
- Missing columns raises ValueError with a clear message.
- Empty query returns a user-facing input error.
- Unexpected failures are logged and reported in the UI.

Interview-Focused Summary

Problem Statement
SemanticShelf enables meaning-based book discovery. It maps a natural-language query and book metadata into the same embedding space, then retrieves the closest matches. This solves the mismatch between user intent and keyword search by capturing semantics and paraphrases.

Dataset and Stats
See "Dataset Details" and "Dataset Size and Statistics" above for schema, cleaning rules, and computed counts.

Data Preparation and Feature Construction
The data loader validates required columns, fills missing values with empty strings, trims whitespace, and creates a combined text feature using title, authors, categories, and description. Rows without a title are dropped to avoid empty embeddings.

Model and Retrieval Strategy
The system uses a sentence-transformers model to create dense embeddings. These vectors are stored in a persistent ChromaDB collection. Query vectors are compared against stored vectors to produce a ranked list.

Indexing and Query Flow
- Indexing: load CSV, build combined text, embed in batches, store embeddings + metadata in ChromaDB.
- Query: embed the input text, run vector similarity search, convert distance to a bounded score, return metadata.

Evaluation Metrics and Results
Metrics for retrieval are Precision@k, Recall@k, MRR, and NDCG. This build does not include labeled relevance data, so numeric results are not reported. When labels are available, measure ranking metrics plus latency, index build time, and index size.

UI Outputs and User Choices
The Streamlit interface provides a query input, a result count slider, a Search button, and a Clear button. Output includes a status message and a results grid. Each card shows title, authors, categories, score, and a truncated description. Empty input, errors, or no matches display clear UI states.

Tech Stack and Rationale
- Python for the overall pipeline and UI.
- Pandas for CSV ingestion and cleaning.
- Sentence-Transformers for semantic embeddings.
- ChromaDB for persistent, fast similarity search.
- Streamlit for a fast, production-ready demo UI.
- python-dotenv for environment-driven configuration.

Key Functions and Their Purpose
- Settings.__post_init__() validates configuration boundaries.
- load_books() enforces schema and builds the combined text feature.
- load_embedding_model() caches the embedding model for reuse.
- get_collection() and reset_collection() manage the persistent vector store.
- BookRecommender.initialize() builds or reuses the index safely.
- BookRecommender.recommend_books() executes query embedding and retrieval.

Performance and Cost Considerations
Batch size controls memory usage. The persistent vector store reduces cold-start cost. The chosen embedding model balances accuracy and latency for local deployments.

Deployment Notes
The app runs with Streamlit. It can be wrapped with FastAPI for API access if required. Dependencies and runtime are pinned for reproducibility.

Challenges and Mitigations
Missing columns and empty text are handled with validation and cleaning. Duplicate identifiers are deduplicated with deterministic suffixes. Cold start is mitigated by persistent storage and model caching.

Future Improvements
Add labeled evaluation, hybrid retrieval (BM25 + embeddings), cross-encoder reranking, personalization signals, and incremental indexing.

Glossary
- Embedding: vector representation of text meaning.
- Vector database: storage and search layer for embeddings.
- Cosine similarity: vector similarity measure.
- Top-k: number of returned results.
- MRR, NDCG: ranking quality metrics.
